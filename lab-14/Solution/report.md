## 2. Теоретическая база

### PostgreSQL (Реляционная база данных)
- **Таблицы и ограничения:** Данные нормализованы и хранятся в виде отношений. Целостность обеспечивается внешними ключами (`FOREIGN KEY`) и правилами ссылочности.
- **Индексы:** Структуры быстрого доступа к строкам таблицы (B-Tree по умолчанию). Предотвращают сканирование всей таблицы (Sequential Scan), переводя операции поиска в фазу индексного поиска (Index Scan).
- **EXPLAIN / EXPLAIN ANALYZE:** Инструменты анализа планов выполнения. `EXPLAIN` показывает расчетные затраты планировщика, а `EXPLAIN ANALYZE` запускает запрос в реальности и выводит фактическое время выполнения на каждом узле.

### MongoDB (NoSQL документо-ориентированная база данных)
- **Документное хранилище:** Данные сохраняются как BSON документы в коллекциях. Схема гибкая и может меняться от документа к документу.
- **Денормализация:** Связанные сущности внедряются непосредственно внутрь родительского документа в виде массивов или подобъектов (например, товары и клиент внутри документа заказа). Это ускоряет операции чтения, но усложняет согласованность при обновлениях.
- **Анализ запросов (Explain Plan):** MongoDB предоставляет метод `explain("executionStats")` для проверки планов поиска и оценки использования индексов в коллекциях.

## 4. Контрольные вопросы

1. **Различия SQL и NoSQL баз данных:**  
   SQL-базы используют строгую реляционную модель, нормализацию таблиц и ACID-транзакции. NoSQL-базы предлагают гибкую схему данных, денормализованные документы, горизонтальное масштабирование и высокую скорость чтения сложных вложенных структур.
2. **Как работает EXPLAIN в PostgreSQL:**  
   Показывает шаги (дерево операций), выбранные планировщиком для выполнения запроса. Выводит оценку стоимости первого/всех элементов (`cost`), количество возвращаемых строк (`rows`) и ширину в байтах (`width`).
3. **Плюсы и минусы денормализации данных:**  
   Плюсы: быстрая выдача данных (нет необходимости в тяжелых операциях JOIN), простота запросов чтения. Минусы: избыточность хранения (данные дублируются), потенциальное нарушение согласованности данных при обновлении (например, смена адреса клиента потребует перезаписи всех его заказов).
4. **Типы индексов в СУБД:**  
   B-Tree (стандартный для сортированных данных), Hash (точный поиск по равенству), GIN (инвертированный для массивов и JSON), GiST (для гео-данных).
