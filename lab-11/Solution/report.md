## 2. Теоретическая база

### FastAPI
- **FastAPI:** Производительный и современный фреймворк для написания API на Python. Базируется на стандартах Starlette и Pydantic. Поддерживает конкурентную обработку за счет асинхронности.
- **Pydantic-модели:** Классы, определяющие ограничения и типы параметров. Используются для валидации данных тела запроса, Query параметров и структуры HTTP ответов.
- **Интерактивная документация:** FastAPI анализирует Pydantic-схемы и роуты, генерируя стандартную JSON-спецификацию OpenAPI, визуализируемую через Swagger UI по пути `/docs`.

### Express.js
- **Express:** Гибкий веб-фреймворк для платформы Node.js, реализующий минималистичный набор инструментов маршрутизации.
- **Middleware:** Функции-обработчики, встраиваемые в конвейер обработки запроса. Могут выполнять предварительную проверку, прерывать обработку или модифицировать объекты запроса/ответа.
- **Joi:** JavaScript-библиотека для декларативного описания и runtime-валидации JSON-объектов.

## 4. Ответы на контрольные вопросы

### Часть 1
1. **Плюсы Pydantic для валидации:**  
   Позволяет описывать структуру данных через декларативные подсказки типов Python. Валидация происходит автоматически в рантайме. Pydantic-модели легко конвертируются в JSON схемы для OpenAPI.
2. **Как собирается документация в FastAPI:**  
   FastAPI рефлексирует типы входящих и исходящих параметров в хэндлерах путей, компилирует схему OpenAPI (в формате JSON), которая затем читается Swagger UI и выводится в интерактивном веб-интерфейсе.
3. **Зачем нужна проверка уникальности ISBN:**  
   ISBN — уникальный код книги. Дублирование ISBN сделает невозможным точное отслеживание конкретного издания книги в библиотечной системе.
4. **Статус-коды ответов:**  
   - `200 OK` — успешное чтение/изменение.
   - `201 Created` — успешная вставка новой записи.
   - `204 No Content` — успешное удаление без тела ответа.
   - `400 Bad Request` — логическая ошибка (попытка удалить книгу, которой нет в наличии).
   - `404 Not Found` — ресурс по ID отсутствует.
   - `409 Conflict` — попытка вставить дублирующийся ISBN.

### Часть 2
5. **Middleware в Express:**  
   Функции, выполняемые по цепочке. Использовались `cors` (кросс-доменные разрешения), `helmet` (безопасность заголовков), `express-rate-limit` (ограничение частоты запросов), а также кастомные валидаторы тела запроса и параметров пути.
6. **Сравнение Joi и Pydantic:**  
   Joi описывает правила императивно в JS-коде, валидацию нужно вызывать вручную (например, в middleware). Pydantic работает на уровне типов Python, глубоко интегрирован в сигнатуру функций FastAPI и отрабатывает автоматически при входе в роут.
7. **Удобство хранения данных в JSON-файле:**  
   Идеально для прототипирования: не требуется развертывание СУБД, а файл легко прочитать и отредактировать вручную.
8. **Оптимизация под Production:**  
   Интеграция полноценной БД (PostgreSQL), внедрение системы логирования (Winston), защита маршрутов (OAuth2/JWT) и упаковка бэкенда в Docker-контейнер.
