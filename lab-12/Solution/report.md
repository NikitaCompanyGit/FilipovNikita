## 2. Теоретическая база

### Progressive Web Apps (PWA)
- **Service Worker:** Фоновый JS-скрипт, функционирующий в браузере отдельно от страницы. Выполняет задачи сетевого проксирования, кэширования статического контента и обеспечения автономного режима работы.
- **Web App Manifest (`manifest.json`):** Файл описания конфигурации веб-приложения для операционной системы, дающий возможность пользователю устанавливать веб-сайт на домашний экран в виде мобильного приложения.
- **IndexedDB:** Транзакционная нереляционная СУБД, встроенная в браузер для надежного хранения больших объемов структурированных данных на стороне клиента.

### Нативная разработка на Android
- **Jetpack Compose:** Современный декларативный фреймворк UI разработки под Android. Вся разметка пишется на Kotlin в виде `@Composable` функций, автоматически реагирующих на изменения передаваемых переменных состояния.
- **Room Persistence Library:** Надстройка над реляционной СУБД SQLite, упрощающая доступ и маппинг таблиц БД в объекты классов Kotlin с помощью аннотаций `@Entity`, `@Dao`, `@Database`.
- **Архитектура MVVM:** Структурирует проект на Model (данные/БД), View (интерфейс Composable) и ViewModel (бизнес-логика, хранение состояния экрана и обработка действий пользователя).

## 3. Практическая реализация

### Часть 1: Лента новостей PWA (NewsAggregator)
Разработано новостное PWA приложение. Внедрен Service Worker, кэширующий основные скрипты и файлы разметки в оффлайн режиме.

#### Код жизненного цикла Service Worker (`sw.js`):
```javascript
const CACHE_NAME = 'filipov-news-cache-v1';
const ASSETS = [
  '/',
  '/index.html',
  '/src/js/app.js',
  '/src/js/db.js',
  '/manifest.json'
];

self.addEventListener('install', event => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(ASSETS))
  );
});

self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(cachedResponse => cachedResponse || fetch(event.request))
  );
});
```

## 4. Контрольные вопросы

### Часть 1
1. **Жизненный цикл Service Worker:**  
   Скрипт проходит этапы регистрации (`register` в основном потоке), установки (`install` — здесь кэшируется статика) и активации (`activate` — удаление старых кэшей). Далее он переходит в состояние ожидания и обрабатывает события `fetch`.
2. **Как работает кэширование ресурсов:**  
   Служба перехватывает все исходящие GET-запросы страницы. Сначала проверяется локальный кэш через `caches.match()`. Если ресурс найден — он возвращается без сетевого запроса, иначе запрашивается из интернета.
3. **Отличия LocalStorage и IndexedDB:**  
   LocalStorage имеет синхронный интерфейс доступа, хранит только строки типа "ключ-значение" и ограничен лимитом в 5 МБ. IndexedDB — асинхронное объектное хранилище с поддержкой индексов, транзакций и практически неограниченным объемом данных.
4. **Требования к установке PWA на экран:**  
   Обязательное наличие HTTPS протокола, валидного файла `manifest.json` со всеми иконками, и зарегистрированного Service Worker с обработчиком события `fetch`.

### Часть 2
5. **Преимущества Compose перед XML:**  
   Код пишется декларативно в рамках одного языка (Kotlin), исключается ручная связка представлений через `findViewById()`, интерфейс автоматически перестраивается при изменении переменных состояния.
6. **Роль Room DB и основные аннотации:**  
   Облегчает доступ к SQLite базе данных. `@Entity` обозначает структуру таблицы, `@Dao` описывает интерфейс запросов к таблице, `@Database` инициализирует само подключение к базе.
7. **ViewModel и StateFlow:**  
   ViewModel сохраняет данные при повороте экрана устройства. StateFlow предоставляет реактивный поток, отслеживаемый Composable-компонентами для перерисовки экрана при обновлении данных.
8. **Что такое Coroutines в Kotlin:**  
   Это библиотека легких потоков для выполнения асинхронных операций. В Android применяется для выноса операций с диском и сетью из главного потока во избежание блокирования графического интерфейса.
