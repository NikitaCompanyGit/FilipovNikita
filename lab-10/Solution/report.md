## 2. Теоретическая база

### Разработка на React с Vite
- **Компонентная модель:** UI разделяется на независимые блоки (функции), которые получают параметры (`props`) и возвращают разметку JSX.
- **Локальное состояние (useState):** React-хук, используемый для отслеживания динамических изменений в данных компонента. При вызове функции-сеттера состояния планируется повторный рендер соответствующей ветки дерева элементов.
- **Принцип неизменяемости (Immutability):** Прямая модификация переменных состояния запрещена. Обновление должно производиться путем создания копий (например, с помощью spread-оператора или методов типа `filter`/`map`).

### Фреймворк Next.js
- **App Router:** Новая файловая система маршрутизации, где пути к страницам определяются вложенными каталогами внутри директории `app`.
- **Серверные компоненты (Server Components):** Позволяют выполнять рендеринг разметки на стороне сервера, уменьшая объем отправляемого клиенту JavaScript-кода.
- **SSG (Static Site Generation):** Генерация статического HTML на этапе компиляции приложения, что повышает производительность и улучшает поисковую оптимизацию (SEO).

### Часть 1: Менеджер задач (todo-app)
Было спроектировано SPA-приложение для ведения списка дел. Описан тип данных `TodoRecord` (`taskId`, `textValue`, `isCompleted`). 

#### Ключевой блок управления задачами (`App.tsx`):
```tsx
// Добавление новой задачи в список
const handleInsertTodo = (text: string) => {
  const newRecord: TodoRecord = {
    taskId: crypto.randomUUID(),
    textValue: text,
    isCompleted: false,
  };
  setItems([...items, newRecord]);
};

// Изменение флага готовности
const handleToggleTodo = (taskId: string) => {
  setItems(
    items.map((item) =>
      item.taskId === taskId ? { ...item, isCompleted: !item.isCompleted } : item
    )
  );
};
```

## 4. Ответы на контрольные вопросы

### Часть 1
1. **Принцип работы useState:**  
   Хук регистрирует локальную переменную состояния. Возвращает кортеж из значения и функции-модификатора. При вызове модификатора React сверяет изменения и перерисовывает компонент.
2. **Зачем сохранять иммутабельность состояния:**  
   React использует поверхностное сравнение ссылок. При мутации старого массива ссылка не изменяется, поэтому React не узнает об обновлении и не инициирует рендер интерфейса.
3. **Метод удаления задач из массива:**  
   Используется метод `filter()`, возвращающий новый массив без целевого элемента, что полностью удовлетворяет правилу неизменяемости.
4. **Роль TypeScript в React:**  
   Обеспечивает валидацию пропсов и типов состояния при компиляции, снижая риск ошибок в рантайме.

### Часть 2
5. **Что такое SSG:**  
   Static Site Generation — предварительная генерация статических HTML-страниц во время компиляции. Это минимизирует нагрузку на бэкенд и ускоряет загрузку страниц.
6. **Маршрутизация в Next.js App Router:**  
   Маршруты строятся на основе структуры папок внутри директории `app`. Наличие файла `page.tsx` в папке делает ее доступной как роут.
7. **Зачем нужна функция generateStaticParams:**  
   Она возвращает список параметров для динамических роутов, чтобы Next.js сгенерировал под них статические HTML-страницы при сборке.
8. **Разница между dev и build командами:**  
   `npm run dev` запускает сервер разработки с отслеживанием изменений (HMR) и сборкой "на лету". `npm run build` создает оптимизированную и сжатую production-сборку проекта.
